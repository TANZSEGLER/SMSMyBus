getstops

Overview

The getstops method will return the geo locations for all stops of a specified route.

URL

http://www.smsmybus.com/api/v1/getstops

Fields

Name Value Description
key String (required) 16-digit API access key
routeID String (required) A two-digit value describing a valid route in the Metro system.
destination String A value describing a valid route endpoint in the Metro system. When a destination is included in the query, the results will be limited to the stops that are headed in that direction.

Examples

Retrieve all stops for route 3:

http://www.smsmybus.com/api/v1/getstops?key=xxx&routeID=03

Retrieve all stops for route 3 headed toward ETP:

http://www.smsmybus.com/api/v1/getstops?key=xxx&routeID=03&destination=ETP

Response

The response length is dependent on the particular request parameters. If the status field is 0, the response will be grouped by route, listing the details of each stop on that route.

No assumptions should be made about the order of the stops in the response.

<?xml version=”1.0”?> <SMSMyBusResponse> <status>0</status> <timestampe>12:38pm</timestamp> <route> <routeID>3</routeID> <stop> <stopID>1391</stopID> <intersection>Atwood & Ohio</intersection> <lat>43.0937715</lat> <lon>-89.3467281</lon> <direction>ETP</direction> </stop> <stop> <stopID>1961</stopID> <intersection>Atwood & Rusk</intersection> <lat>43.0924936</lat> <lon>-89.3528187</lon> <direction>ETP</direction> </stop> ... ... </route> </SMSMyBusResponse>

Error

An error is indicated by a non-zero status value. If the status value is non-zero, the description field will be present describing the error in plain English.

<?xml version=”1.0”?> <SMSMyBusResponse> <status>-1</status> <description>Invalid request parameters</description> </SMSMyBusResponse>

getnearbystops

Overview

The getstops method will return the geo locations for all stops near a specified location.

URL

http://www.smsmybus.com/api/v1/getnearbystops

Fields

Name Value Description
key String (required) 16-digit API access key
lat String (required) A geo-spaticial figure describing the latitude coordinate for a Metro stop.
lon String (required) A geo-spaticial figure describing the longitude coordinate for a Metro stop.
radius Integer The radius around the lat and lon parameters where you would like to search. The radius is in feet and must be less than 5000. It defaults to 1500.
routeID String A two-digit value describing a valid route in the Metro system. If a routeID is provided, the results will be limited to stops that service that route.
destination String A value describing a valid route endpoint in the Metro system. When a destination is included in the query, the results will be limited to the stops that are headed in that direction.

Examples

Retrieve all stops for route 3 around a point on Atwood (43.0733673,-89.4014633):

http://www.smsmybus.com/api/v1/getnearbystops?key=xxx&lat=43.0733673&lon=-89.4014633&routeID=03

Response

The response length is dependent on the particular request parameters. If the status field is 0, the response will be grouped by route, listing the details of each stop on that route within the specified radius.

No assumptions should be made about the order of the stops in the response.

<?xml version=”1.0”?> <SMSMyBusResponse> <status>0</status> <timestampe>12:38pm</timestamp> <stop> <stopID>1391</stopID> <intersection>Atwood & Ohio</intersection> <lat>43.0937715</lat> <lon>-89.3467281</lon> <direction>ETP</direction> </stop> <stop> <stopID>1961</stopID> <intersection>Atwood & Rusk</intersection> <lat>43.0924936</lat> <lon>-89.3528187</lon> <direction>ETP</direction> </stop> ... ... </SMSMyBusResponse>

Error

An error is indicated by a non-zero status value. If the status value is non-zero, the description field will be present describing the error in plain English.

<?xml version=”1.0”?> <SMSMyBusResponse> <status>-1</status> <description>Invalid request parameters</description> </SMSMyBusResponse>

getvehicles

Overview

The getvehicles method will return the geo locatios for all buses currently driving a specified route.

URL

http://www.smsmybus.com/api/v1/getvehicles

Fields

Name Value Description
key String (required) 16-digit API access key
routeID String (required) A two-digit value describing a valid route in the Metro system. If a routeID is provided, the results will be limited to stops that service that route.
direction String A value describing a valid route endpoint in the Metro system. When a destination is included in the query, the results will be limited to the stops that are headed in that direction.
vehicleID String A three-digit value describing a valid vehicle (bus) in the Metro system and currently driving a route. The vehicleID filter will only be applied to the specified routeID parameter.

Examples

Retrieve all stops for route 3 around a point on Atwood (43.0733673,-89.4014633):

http://www.smsmybus.com/api/v1/getnearbystops?key=xxx&routeID=03

Response

The response length is dependent on the particular request parameters. If the status field is 0, the response will be grouped by route, listing the details of each stop on that route within the specified radius.

No assumptions should be made about the order of the stops in the response.

<?xml version=”1.0”?> <SMSMyBusResponse> <status>0</status> <timestampe>12:38pm</timestamp> <vehicle> <vehicleID>994</vehicleID> <routeID>03</routeID> <next-intersection>Atwood & Ohio</next-intersection> <next-stopID>1391</next-stopID> <lat>43.0937715</lat> <lon>-89.3467281</lon> <direction>ETP</direction> </vehicle> <vehicle> <vehicleID>145</vehicleID> <routeID>03</routeID> <next-intersection>Atwood & Rusk</next-intersection> <next-stopID>1712</next-stopID> <lat>43.0924936</lat> <lon>-89.3528187</lon> <direction>WTP</direction> </vehicle> ... ... </SMSMyBusResponse>

Error

An error is indicated by a non-zero status value. If the status value is non-zero, the description field will be present describing the error in plain English.

<?xml version=”1.0”?> <SMSMyBusResponse> <status>-1</status> <description>Invalid request parameters</description> </SMSMyBusResponse>
